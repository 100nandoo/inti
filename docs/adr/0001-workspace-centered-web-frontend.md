# Workspace-centered web frontend

Inti will treat the main web app as a text workspace with one canonical **Working Text** instead of a four-step wizard with separate editable OCR, summary, and TTS text buffers. OCR and summarization produce non-destructive **Transform Results** that users can promote by replacing or appending the working text, with per-transform default promotion behavior stored in settings, while speech generation operates on working text or the latest text result and produces a separate **Audio Result** that survives later text edits. We chose this because it is a hard-to-reverse state and layout decision, the current multi-panel flow is ambiguous enough that future readers would not know which behavior was intentional, and the main alternative of preserving multiple peer text buffers would keep reintroducing contradictory sources of truth.
